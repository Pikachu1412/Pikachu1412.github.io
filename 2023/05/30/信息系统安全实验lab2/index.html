任务一：chroot

子任务1

  1. 首先make

  2. 为 touchstone 程序添加 setuid root 权限，并启动执行

  3. 在/tmp 目录下面创建/tmp/test.txt 文件，并将其 owner 改成 root

  4. 修改 exploit 代码，尝试利用 touchstone 的漏洞，删除/tmp/test.txt文件

    查看base基地址0xf7c00000

​ 查看system地址

​ 查看exit地址

​ 查看unlink地址

​ 查看”/bin/sh”地址

​ 查看ebp基址

将以上地址都修改后运行exp1.py python3 exp1.py 127.0.0.1 80

发现/tmp/test.txt已经被删除

子任务2

在server.c中添加如下代码

重新make

chmod +x chroot-setup.sh chroot-copy.sh

sudo ./chroot-setup.sh

cd /jail

sudo ./touchstone

sudo gdb

attach 407652

发现基址没有变化

执行攻击脚本，发现/tmp下的test没有被删除，只删除了/jail/tmp下的test

任务 2： 改变进程 euid

在server.c中添加代码setresuid(1000,1000,1000);即将权限设置为普通用户的权限

make

运行touchstone和攻击脚本，发现test.txt没有被删除，因为普通用户无法删除root权限级的文件
